HTML

  • "" and other declarations are not case-sensitive but html elements etc. are
  • Webpages need to know the website is using html language which is what the declaration does.
  • Changes can be made to html documents, and updated through the creation of a new branch on your repository using terminal (mac)
  • It typically consists of three sections: a head, body and footer
  • Some are sematic html elements i.e. they function and hold content according to their name e.g. body holds the body of the webpage content. Section is another element
  • Head holds metadata and meta tags that allow retrieval of information from external sources that allows for better performance, search engine optimisation (i.e. better ranking in google search/bing/safari

Git

  • Git is the version control system that we use to connect local work to the online cloud-system Github.
  • You can navigate between branches using commands such as git checkout main and git checkout -b feature/_____ . The latter creates a new branch.
  • Always check which branch you are in prior to and after using commands to make sure further alterations and additions are made to the correct working branch!
  • To do this, use command - git status

CSS styles

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
